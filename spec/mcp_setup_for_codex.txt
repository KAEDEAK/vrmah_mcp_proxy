# VRM MCP Proxy - Codex CLI 設定手順

> **[環境確認]** このドキュメントは **Codex CLI** 用です。
> 作業前に自分の実行環境を確認してください:
> - VSCode 拡張として動作中 → `mcp_proxy/spec/mcp_setup_for_vscode.txt` を参照
> - Claude Code CLI として動作中 → `mcp_proxy/spec/mcp_setup_for_claude.txt` を参照
> - Codex CLI として動作中 → **このドキュメント**

## 概要
VRM Agent Host と VOICEVOX を Model Context Protocol (MCP) ツールとして
Codex CLI から利用できるようにするための設定手順です。

`mcp_proxy/mcp_server.py` が以下の機能を提供:
- VRM Agent Host HTTP API の MCP ラッピング
- VOICEVOX 音声合成 + VRM Agent Host への音声転送

対応MCPプロトコルバージョン: 2025-11-25（現行実装）

## 前提条件
- Python 3.x がインストールされていること
- `python3` または `python` または `py` コマンドが利用可能であること
- requests ライブラリがインストールされていること (`pip install requests`)
- VRM Agent Host サーバーが稼働していること
- Codex CLI がインストールされていること（`codex mcp` が使えるバージョン）
- (オプション) VOICEVOX がインストール・稼働していること

## 設定手順

### 0. config.json の設定

`mcp_proxy/config.json` を環境に合わせて編集:

```json
{
  "version": 2.0,
  "vrmah": {
    "host": "http://<ip-address-vrmah>:34560",
    "candidates": ["http://<ip-address-vrmah-candidate>:34560"]
  },
  "voice": {
    "type": "voicebox",
    "server": "http://<ip-address-voicevox>:50021",
    "candidates": ["http://<ip-address-voicevox-candidate>:50021", "http://<ip-address-vrmah>:50021"],
    "name": "櫻歌ミコ",
    "speaker_uuid": "0693554c-338e-4790-8982-b9c6d476dc69",
    "style_id": 43
  }
}
```

設定項目:
- `version`: 設定ファイルのバージョン（現在 `2.0`）
- `vrmah.host`: VRM Agent Host のプライマリ URL
- `vrmah.candidates`: フォールバック先の URL リスト（オプション）。プライマリに接続できない場合、リスト順に試行する。成功した候補はプライマリに昇格する
- `voice.type`: 現在は `voicebox` のみ対応
- `voice.server`: VOICEVOX サーバーのプライマリ URL
- `voice.candidates`: VOICEVOX のフォールバック先 URL リスト（オプション）。動作は `vrmah.candidates` と同様
- `voice.style_id`: デフォルトで使用するスピーカーID

スピーカーIDは `voicevox_speakers` ツールで確認可能。

注: 旧形式（`"vrmah": "http://..."` の文字列指定）も後方互換として動作しますが、candidates は使用できません。

### 1. MCPサーバーの登録

プロジェクトルートで以下のコマンドを実行:

```bash
codex mcp add vrm_proxy_codex -- python3 mcp_proxy/mcp_server.py
```

パラメータ説明:
- `vrm_proxy_codex`: MCPサーバー名
- `-- python3 mcp_proxy/mcp_server.py`: 実行コマンド（`--` 以降がサブプロセス）

補足:
- `python3` がない環境では `python` または `py` に置き換える
- VRM Agent Host / VOICEVOX の接続先は `config.json` で設定

### 2. 登録確認

```bash
codex mcp list
codex mcp get vrm_proxy_codex
```

### 3. 設定ファイルの確認

Codex CLI の MCP設定は `~/.codex/config.toml` に保存されます。
登録後、`[mcp_servers.vrm_proxy_codex]` セクションが追加されます。

例:

```toml
[mcp_servers.vrm_proxy_codex]
command = "python3"
args = ["mcp_proxy/mcp_server.py"]
```

## 使い方

### Codex セッション再起動

MCPサーバー登録後に Codex セッションを再起動すると、以下のツールが利用可能になります:

1. **vrm_command**: 単一のVRM HTTPコマンドを送信
   - パラメータ:
     - `target`: VRM APIのターゲット（例: `animation`）
     - `cmd`: コマンド名（例: `play`）
     - `params`: 追加パラメータ（例: `{"id": "Idle_cute"}`）
     - `method`: HTTPメソッド（デフォルト: `GET`）
     - `timeout`: タイムアウト秒数（デフォルト: `10.0`）
     - `headers`: HTTPヘッダー（オプション）
     - `json_payload`: POST/PUT用JSONボディ（オプション）
     - `text_payload`: POST/PUT用テキストボディ（オプション）
     - `absolute_url`: ベースURLの完全上書き（上級者向け）

2. **batch_vrm_commands**: 複数のVRMコマンドを順次実行
   - パラメータ:
     - `commands`: `vrm_command` 形式のコマンド配列

3. **voicevox_speak**: VOICEVOXで音声合成しVRM Agent Hostで再生
   - パラメータ:
     - `text`: 読み上げるテキスト (必須)
     - `speaker_id`: スピーカーID (省略時: `config.json` の `style_id`)
     - `speed_scale`: 速度 (デフォルト: `1.0`)
     - `volume_scale`: 音量 (デフォルト: `1.0`)

4. **voicevox_speakers**: 利用可能なスピーカー一覧を取得

補足:
- `vrm_command` / `batch_vrm_commands` は常に利用可能です。
- `voicevox_speak` / `voicevox_speakers` は `config.json` の VOICEVOX 設定が正しく、VOICEVOX サーバー接続に成功した場合のみ利用可能です。

## 留意事項

- `mcp_proxy/mcp_server.py` は **最初に受け取った形式でフレーミング方式を自動判定**します。
  - NDJSON（1行1JSON）で最初のメッセージを送ると NDJSON 方式として固定されます。
  - Content-Length 方式で送る場合は、ヘッダ/本文の読み取り実装が一致している必要があります。
  - 簡易テストでは NDJSON で送るほうが安定しやすいです。

## 使用例

### アニメーション再生

```text
vrm_command: target=animation, cmd=play, params={"id": "Idle_cute", "seamless": "y"}
```

### VOICEVOX 発話

```text
voicevox_speak: text="作業完了しました"
```

## トラブルシューティング

### MCPサーバーが接続できない場合

1. VRM Agent Hostが稼働しているか確認:
```bash
curl "http://<ip-address-vrmah>:34560/?target=animation&cmd=play&id=Idle_cute"
```

2. Pythonスクリプトが直接動作するか確認:
```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' | python3 mcp_proxy/mcp_server.py
```
`python3` がない環境では次も可:
```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' | python mcp_proxy/mcp_server.py
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' | py mcp_proxy/mcp_server.py
```

3. MCPサーバーを削除して再登録:
```bash
codex mcp remove vrm_proxy_codex
codex mcp add vrm_proxy_codex -- python3 mcp_proxy/mcp_server.py
```
`python3` がない環境では `python` または `py` に置き換えてください。

### ツールが表示されない場合

Codex セッションを完全に再起動してください。既存セッションでは新規登録したMCPサーバーを認識しないことがあります。

### ベースURLを変更したい場合

`mcp_proxy/config.json` の `vrmah.host` 値を変更し、Codex セッションを再起動してください。フォールバック先も変更する場合は `vrmah.candidates` を編集してください。

## Codex MCP コマンドリファレンス

```bash
# MCPサーバー一覧
codex mcp list

# MCPサーバー追加 (stdio)
codex mcp add <name> -- <command> [args...]

# MCPサーバー追加 (streamable HTTP)
codex mcp add <name> --url <url>

# MCPサーバー詳細
codex mcp get <name>

# MCPサーバー削除
codex mcp remove <name>
```
