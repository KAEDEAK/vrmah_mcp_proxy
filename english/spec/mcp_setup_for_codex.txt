# VRM MCP Proxy - Codex CLI Setup Guide

> **Environment check**: This document is for **Codex CLI**.
> Confirm your runtime before setup:
> - VS Code extension -> see `mcp_proxy/spec/mcp_setup_for_vscode.txt`
> - Claude Code CLI -> see `mcp_proxy/spec/mcp_setup_for_claude.txt`
> - Codex CLI -> **this document**

## Overview
This guide explains how to use VRM Agent Host and VOICEVOX as MCP tools from Codex CLI.

`mcp_proxy/mcp_server.py` provides:
- MCP wrapper for VRM Agent Host HTTP API
- VOICEVOX synthesis + audio handoff to VRM Agent Host

Supported MCP protocol version: `2025-11-25` (current implementation)

## Prerequisites
- Python 3.x installed
- One of `python3`, `python`, or `py` available
- `requests` installed (`pip install requests`)
- VRM Agent Host server running
- Codex CLI installed (version with `codex mcp`)
- Optional: VOICEVOX installed and running

## Setup

### 0. Configure `config.json`

Edit `mcp_proxy/config.json`:

```json
{
  "version": 2.0,
  "vrmah": {
    "host": "http://<ip-address-vrmah>:34560",
    "candidates": ["http://<ip-address-vrmah-candidate>:34560"]
  },
  "voice": {
    "type": "voicebox",
    "server": "http://<ip-address-voicevox>:50021",
    "candidates": ["http://<ip-address-voicevox-candidate>:50021", "http://<ip-address-vrmah>:50021"],
    "name": "Ouka Miko",
    "speaker_uuid": "0693554c-338e-4790-8982-b9c6d476dc69",
    "style_id": 43
  }
}
```

Fields:
- `version`: config file version (currently `2.0`)
- `vrmah.host`: primary VRM Agent Host URL
- `vrmah.candidates`: fallback URL list (optional). Tried in order when the primary is unreachable. A successful candidate is promoted to primary
- `voice.type`: currently `voicebox` only
- `voice.server`: primary VOICEVOX server URL
- `voice.candidates`: VOICEVOX fallback URL list (optional). Same behavior as `vrmah.candidates`
- `voice.style_id`: default speaker ID

Note: The legacy format (`"vrmah": "http://..."` as a plain string) still works for backward compatibility, but candidates cannot be used.

### 1. Register MCP server

Run from project root:

```bash
codex mcp add vrm_proxy_codex -- python3 mcp_proxy/mcp_server.py --base-url http://<ip-address-vrmah>:34560
```

Notes:
- Replace `python3` with `python` or `py` if needed
- VRM Agent Host and VOICEVOX targets are read from `config.json`

### 2. Verify registration

```bash
codex mcp list
codex mcp get vrm_proxy_codex
```

### 3. Check config file

Codex CLI stores MCP settings in `~/.codex/config.toml`.
After registration, `[mcp_servers.vrm_proxy_codex]` is added.

Example:

```toml
[mcp_servers.vrm_proxy_codex]
command = "python3"
args = ["mcp_proxy/mcp_server.py"]
```

## Usage

Restart the Codex session after registration.
Then the following tools become available:

1. **vrm_command**: send one VRM HTTP command
2. **batch_vrm_commands**: run multiple VRM commands sequentially
3. **voicevox_speak**: synthesize speech and play in VRM Agent Host
4. **voicevox_speakers**: list available speakers

### Note on framing
`mcp_proxy/mcp_server.py` auto-detects framing from the first message.
- If first message is NDJSON, it stays in NDJSON mode
- If first message uses Content-Length framing, client header/body parsing must match
- NDJSON is usually easier for quick manual tests

## Examples

### Play animation

```text
vrm_command: target=animation, cmd=play, params={"id": "Idle_cute", "seamless": "y"}
```

### VOICEVOX speech

```text
voicevox_speak: text="Task completed"
```

## Troubleshooting

### MCP server cannot connect

1. Check VRM Agent Host:
```bash
curl "http://<ip-address-vrmah>:34560/?target=animation&cmd=play&id=Idle_cute"
```

2. Check Python script directly:
```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' | python3 mcp_proxy/mcp_server.py
```

3. Remove and re-add:
```bash
codex mcp remove vrm_proxy_codex
codex mcp add vrm_proxy_codex -- python3 mcp_proxy/mcp_server.py --base-url http://<ip-address-vrmah>:34560
```

### Tools do not appear

Completely restart the Codex session.

### Change base URL

Edit `vrmah.host` in `mcp_proxy/config.json`, then restart Codex. To change fallback URLs, edit `vrmah.candidates`.

## Codex MCP Command Reference

```bash
# List MCP servers
codex mcp list

# Add MCP server (stdio)
codex mcp add <name> -- <command> [args...]

# Add MCP server (streamable HTTP)
codex mcp add --transport streamable-http <name> <url>

# Show MCP server details
codex mcp get <name>

# Remove MCP server
codex mcp remove <name>
```
