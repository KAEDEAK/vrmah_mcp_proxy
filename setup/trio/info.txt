# 3体制御する例

このディレクトリに入っているバッチファイルは、VRM Agent Host を 3体起動してある状況で、VRMをロードしてウィンドウ位置・サイズを変更しVRMを配置する例です。

単純に VRM Agent Host を３つのフォルダにコピーして、config.json または override.json で ポート番号を変更し、別々に起動してください。
VRM Agent Host の設定例
port
34560 claude
34562 codex
34564 gemini

起動した状態で順に実行すると VRM を読み込んだ状態で配置が完了します。
なお、2体とか1体の状態でも、ただのバッチファイルなので curl の呼び出しでエラーになるだけで、他は機能します。
また、mcp proxy のほうで、想定する VRM Agent Host のサーバーが見つからない場合、別のポートにフォールバックするようにしておけば、複数体の時はそれぞれを使い、1体しか起動してないときは全ての操作を1体に集約することができます。

1.  vrm_trio_load.bat 
2.  vrm_trio_init.bat

こういう感じの配置になります。
https://x.com/KAEDE_AMANOKAWA/status/2037993448628998420


## vrm_trio_load.bat 

このバッチファイルは、以下のポートの VRMAH に対して、異なるモデルを読み込む例です。
ファイル名はご自身で配置されたものへ変更してください。

34560 sample01.vrm
34562 sample05.vrm
34564 sample02.vrm


## vrm_trio_init.bat

３体に対してデスクトップ上のウィンドウ位置・サイズ、ワールド座標上のアバター位置・角度の調整を行います。
2K モニター用です。2K モニターでない場合は、適宜座標を変えてください。

例 curl -s "http://%HOST%:34564?target=server&cmd=resize_move&left=888&top=606&width=679&height=787"
これらの部分がウィンドウ位置、サイズになります。
left=888
top=606
width=679
height=787


面倒な場合は、Claudeや CodexなどのAIにmcp が登録されている状態で以下のような依頼を行うと作成してもらえます。

```
- これらの現在のデスクトップ上の座標、サイズ、アバター(VRM)のワールド座標、アバターの角度を取得して curl で 再現するバッチファイルを作ってください
- VRM Agent Host を3体、それぞれ (your local ip):34560,34562,34564 で起動しています。
- ユーザーが VRM Agent Host を事前に起動し、そのバッチファイルを使って設定する流れになるので、.exe の起動はしない。
- カメラ設定は不要です。カメラ関連コマンドは使わない
- nul にリダイレクトしない
```

## 補足

なお、カメラ操作はデフォルトでアバター方向を向いているので不要です。
